웹 페이지가 느리게 느껴지는 이유는 네트워크만이 아닙니다. 화면은 이미 보이는데 클릭이나 스크롤이 늦게 반응한다면 브라우저 메인 스레드가 긴 JavaScript 작업에 묶여 있을 가능성이 큽니다. 이런 작업을 Long Task라고 부릅니다.

일반적으로 50ms 이상 메인 스레드를 점유하는 작업은 사용자 입력을 밀어낼 수 있습니다. 성능 개선은 번들 크기를 줄이는 것에서 끝나지 않고, 실행 시간을 작은 단위로 나누는 데까지 이어져야 합니다.

## 먼저 긴 작업을 관측한다

감으로 코드를 줄이기보다 브라우저 Performance 패널이나 RUM 데이터를 통해 실제 Long Task를 찾아야 합니다. 개발자 PC에서는 빠르게 보이는 코드도 저사양 기기에서는 길게 실행될 수 있습니다.

확인할 지표는 다음과 같습니다.

- 긴 task의 발생 시점
- 입력 이벤트 직전의 메인 스레드 점유
- hydration, 렌더링, JSON 파싱, 차트 초기화의 실행 시간
- 특정 라우트나 컴포넌트에서 반복되는 패턴

필드 데이터가 있다면 기기 등급, 브라우저, 페이지 종류별로 나눠보는 것이 좋습니다.

## 초기 로딩 작업을 줄인다

초기 진입 시점에는 사용자가 아직 상호작용하기 전이라 많은 작업을 몰아넣기 쉽습니다. 하지만 이 구간이 길어지면 첫 입력이 늦어집니다.

다음 작업은 지연하거나 분리할 후보입니다.

- 화면 아래 영역의 차트 초기화
- 사용자가 열기 전 모달의 복잡한 렌더링
- 대용량 JSON 파싱
- 에디터, 지도, 그래프 라이브러리 로딩
- 즉시 필요 없는 개인화 추천 계산

첫 화면에 반드시 필요한 작업과 나중에 해도 되는 작업을 분리해야 합니다.

## 큰 작업은 쪼갠다

한 번에 300ms 걸리는 작업을 30ms 단위로 나누면 브라우저가 중간에 입력을 처리할 기회를 얻습니다. 데이터 목록 가공, 문서 파싱, 대량 DOM 업데이트 같은 작업에서 효과가 큽니다.

```js
async function processInChunks(items, handle) {
  for (let i = 0; i < items.length; i += 100) {
    items.slice(i, i + 100).forEach(handle);
    await new Promise((resolve) => setTimeout(resolve, 0));
  }
}
```

이 방식은 총 실행 시간을 줄이지는 않을 수 있지만, 사용자가 느끼는 멈춤을 줄입니다. 계산 자체가 무겁다면 Web Worker도 검토해야 합니다.

## 렌더링 범위를 제한한다

상태 하나가 바뀔 때 큰 트리가 다시 렌더링되면 긴 작업이 생깁니다. 프레임워크를 쓰더라도 렌더링 범위 관리는 필요합니다.

- 자주 바뀌는 상태를 상위 전역 상태에 올리지 않는다.
- 긴 목록은 virtualization을 적용한다.
- 입력 중에는 비싼 필터링을 debounce하거나 transition으로 낮춘다.
- 차트나 에디터는 탭이 활성화될 때 초기화한다.

핵심은 "변경된 부분만 일하게 만들기"입니다.

## 성능 예산을 PR 기준으로 만든다

Long Task는 한 번 개선하고 끝나는 문제가 아닙니다. 새 기능이 들어오면 다시 늘어납니다. 라우트별로 허용 기준을 정하고 PR에서 확인해야 합니다.

```text
상품 목록: 100ms 이상 Long Task 0개
대시보드: 200ms 이상 Long Task 0개, 50ms 이상 5개 이하
에디터: 입력 중 50ms 이상 Long Task 금지
```

모든 페이지에 같은 기준을 적용하기보다 사용자의 작업 흐름에 맞춰 예산을 세우는 것이 현실적입니다.

## 체크리스트

- Performance 패널과 필드 데이터로 Long Task를 먼저 찾는다.
- 초기 로딩에 필요 없는 작업을 지연한다.
- 대량 처리 작업은 작은 chunk로 나눈다.
- 무거운 계산은 Web Worker를 검토한다.
- 상태 변경이 큰 렌더링으로 번지지 않게 한다.
- 라우트별 Long Task 예산을 PR 기준으로 둔다.

사용자가 빠르다고 느끼는 앱은 다운로드가 빠른 앱만이 아닙니다. 클릭했을 때 바로 반응할 수 있도록 메인 스레드에 숨 쉴 틈을 주는 앱입니다.
